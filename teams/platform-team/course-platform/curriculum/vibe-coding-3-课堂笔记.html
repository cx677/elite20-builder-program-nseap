Vibe Coding 第3课

智能体、Skills 与元认知方法论

🎯 AI+X 的三个学习目标

本课程不是教具体知识的，而是教你掌握方法。核心是三个维度：

1️⃣ 世界建模 / 领域建模

任何新领域，先建立它的概念树状结构——有哪些实体、它们之间什么关系。就像学软件工程要先知道数据结构、算法、数据库等主干课程的关系。

"对一个领域形成一个3万英尺的鸟瞰图，你就知道出问题该找哪个方向。"

2️⃣ 人机配合 / 协作模型

怎么跟 AI 有效交互。知道 AI 能干什么、不能干什么，怎么问问题、怎么迭代。

"所有有正确答案的问题，AI 早就学会了。我们要解决的是没有清晰边界的真实问题。"

3️⃣ 元认知

对自己思考的思考——how to learn how to learn。认识到自己在想什么、怎么学得更快、怎么构建能创造技能的技能。

"笛卡尔说我思故我在——人之所以为人，就是能意识到自己在思想。"

🏆 Talent Code：成才三要素

  • 刻意练习 Deliberate Practice — 不是刷题，而是每次挑战能力边缘，在犯错中学习。
  • 大师级教练 Master Coach — AI 是这时代最好的教练：什么都懂、无限耐心、随时反馈。
  • 激情 Passion — 内在驱动力。没有 passion，前面两个无从落地。

"犯错是最好的学习信号——如果一件事你不会犯错，说明已经没有学习机会了。"

🤖 Agent 四层架构

  1. LLM 直接对话 — 最简单的问答模式
  2. RAG（检索增强生成）— 给大模型搭知识库，做更精准的回答
  3. Agent（智能体）— 能调用工具、有记忆、有 Skills，能主动执行任务
  4. Multi-agent Teams — 多智能体编排协作，形成复杂工作流

本节课聚焦第三层——Agent，特别是 Skills 系统。

🧠 Agent 核心概念

🛠 Skills（最重要）

Skill 是智能体的核心能力。Skill 可以调用其他 Skill（Skill-eating-Skill），形成递进式能力链。Skills 是动态生成和自我总结的——Hermes 的 Skills 就是这种机制的典型实现。

💾 Memory（记忆）

Agent 要成长必须有记忆。分两类：

  • 短期记忆（Context）— 每次对话的上下文窗口
  • 长期记忆（Long-term）— 跨 session 记住用户偏好、经验、知识

介绍了 Memory Palace 方法：像把记忆放在房间的不同格子里，结构化存储。

🌐 Ontology（本体论）

"任何事物都可以归结为两件事：结构和过程。"

本体论包含三个层次：

  • 实体（Entity）— 具体的人事物，比如"学生张三"
  • 关系（Relationship）— 实体之间的联系
  • 规律（Law/Rule）— 支配实体和关系的因果律

"你先画好 ontology，再用 skill 去操作它——就像曲奇饼模具和面团的关系。"

🔄 S-T-C-A 框架

任何问题都可以用这个循环解决：

Situation（当前情形）→ Task（目标任务）→ Knowledge（先验知识）→ Action（行动）→ Result（结果）

结果和预期不一致 = 学习机会 → 改进 Knowledge → 下次预测更准。

📐 核心方法论

领域学习的正确姿势

  1. 拿到一个新领域 → 先用 AI 生成 Concept Graph / Mind Map
  2. 从 3 万英尺高度建立整体认知，知道不同概念之间的关系
  3. 根据需求深入细节，而不是从头到尾通读
  4. 用 降维解释法：不懂就问 AI "用小学生能听懂的方式给我讲"

学习的四个环节

  • 动手做（做是最好的学）
  • 犯错 + 反馈（犯错是学习的信号）
  • 循环迭代（S-T-C-A 循环）
  • 提炼复用（把经验固化为 Skill）
💡 "能用 AI 做的事情一定不要手动去做。做完以后，不是记住步骤，而是记住思路。写 AI 日志——你怎么问的、它怎么答的、你哪里不满意——比结果本身更重要。"

Knowledge 的杠杆效应

人的 K（知识）是有限的，但 AI 的 K 是星辰大海。你的能力不取决于你知道多少，而取决于你能调动多少 AI 的知识。

"以前你要花 18 年寒窗积累的知识，现在一句提示词就能调出来。"
