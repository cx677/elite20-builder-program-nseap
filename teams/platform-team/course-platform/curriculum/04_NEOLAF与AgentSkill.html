贯穿整个 C4 系列的核心思想是：你不是在"写代码"，而是在"构造技能"。每个挑战的产出，都是一个可复用的 Agent Skill。

🔁 KSTAR ↔ NEOLAF 对应关系

KSTAR 学习循环的 5 个阶段，对应 NEOLAF Agent 的不同能力层：

KSTAR 阶段 NEOLAF 对应 Skill 角色
K — Knowledge Language Stack (认知层) 构建知识表示
S — Strategy Meta-Language (策略层) 定义行动策略
T — Test Action + Plan (执行层) Agent Skill 在此处执行
A — Assess Feedback Loop 评估结果 → 迭代
R — Reflect Learn / Evolve skill 进化
🔑

核心洞察：Agent Skill 就是 KSTAR 的"行动计划"在 NEOLAF 框架中的具体实现。没有 skill，计划永远停留在策略层，无法落地。

🧠 Hermes Agent ≈ NEOLAF Agent

Hermes Agent 的架构设计与 NEOLAF Agent 在多处不谋而合：

  • 元技能驱动：Hermes 通过 system prompt 注入元技能（如 file operations、code execution），NEOLAF 通过 meta-language 定义 agent 行为
  • 工具调用 (Tool Use)：Hermes 的 function calling 机制对应 NEOLAF 的 action space
  • 可扩展 skill 系统：Hermes Agent 支持加载自定义 .agents/skills/ 目录，这正是 NEOLAF "可插拔能力单元"的实现
  • 迭代循环：Hermes 的思考-行动-观察循环与 NEOLAF 的 plan-act-observe 循环一致

所以 Richard 说："如果你会用 Hermes Agent，你就已经理解了 NEOLAF Agent 的核心设计哲学。"

🔄 元技能生成技能：Richard 的核心方法论

我的所有技能都是通过这种或类似的元技能所生成的。如果我能做到的话，任何小学生都应该能够做到。你也能行！只需尝试一下即可。

Richard · Elite20 Builder Program · C4 系列挑战

这是 C4 系列最重要的精神：Skill 开发不是精英专属。Richard 强调他的所有 skill 都是用"元技能"（meta-prompt / meta-skill）生成的——这意味着：

  • 你不需要从零开始写 skill
  • 你只需要描述你要做什么，然后用元技能去生成
  • 任何小学生都能做到——你绝对可以
🚀

什么是 NEOLAF

📖 NEOLAF = Meta-Language Stack

NEOLAF 是一个元语言栈架构——一种用于构造 AI Agent 行为的层次化框架。它的核心思想是：用语言定义 Agent 的行为，而不是用代码硬编码。

NEOLAF 的全称可以理解为"Natural-language-driven, Extensible, Operational Language Agent Framework"——一种自然语言驱动、可扩展的操作型语言 Agent 框架。

NEOLAF 定义了四层元语言栈，从底层语言基础到顶层行为约束，每层都建立在前一层之上：

1
底层 · 语言栈

Language Stack

Agent 的"母语"——即它使用的自然语言（中文、英文等）和编程语言（Python、TypeScript 等）。这是所有上层能力的基础载体。

示例：Hermes Agent 用中文交流、用 Python 执行代码

2
策略层 · 元语言

Meta-Language

用于描述"Agent 应该如何思考和行动"的语言。包括 system prompt、指令模板、行为约束等。元语言不直接执行任务，而是定义任务应该如何被理解和执行。

示例：Hermes Agent 的 system prompt 就是一段元语言

3
执行层 · Skill

Skill (Agent Skill)

可复用的能力单元——由元语言定义、由语言栈执行。Skill 是 NEOLAF 的"可执行知识"，它封装了完成特定任务所需的指令、工具调用逻辑和行为模式。

示例：一个"文件采集器"skill 封装了读取目录、识别文件类型、提取内容的能力

4
约束层 · 行动计划

Action Plan (行动计划)

由多个 Skill 组合而成的高阶工作流。这是 KSTAR 循环中"行动计划"的直接对应。行动计划定义了用什么 skill、按什么顺序、在什么条件下执行。

示例：C4A 评估器 = 文件采集 skill → 完整性检查 skill → 质量评审 skill → 报告生成 skill

💡

关键理解：NEOLAF 不是一个具体的软件或工具，而是一个设计模式——它告诉我们：好的 Agent 行为不是写死在代码里的，而是用"语言"一层一层定义出来的。每层都是可替换、可扩展的。

🧩

什么是 Agent Skill

🎯 定义：Skill 是可复用的 AI 能力单元

Agent Skill 是 NEOLAF 框架中最重要的一层。它本质上是一个结构化的提示词包，包含了：

  • 角色定义：这个 skill 负责什么任务
  • 指令集：完成任务的具体步骤和规则
  • 工具绑定：需要调用哪些工具（文件读写、代码执行、API 调用等）
  • 输出规范：结果应该以什么格式呈现
  • 边界条件：什么情况下该 skill 生效/不生效

📁 Skill 的文件结构

在 Hermes Agent 或 AG2 系统中，一个 skill 通常是一个目录或一个文件：

.agents/skills/ ├── file-collector/ │ ├── skill.yaml # skill 元数据 │ └── instructions.md # 核心指令 ├── quality-reviewer/ │ ├── skill.yaml │ └── instructions.md └── report-generator/ ├── skill.yaml └── instructions.md

或者在某些系统中，skill 就是一个.mdc 文件（Markdown Context 文件），直接嵌入 Agent 的上下文。

🔄 Skill vs. Prompt 的区别

很多学员会问：Skill 不就是一段提示词吗？

维度 普通 Prompt Agent Skill
复用性 每次手动复制粘贴 自动加载，版本管理
工具绑定 无 / 松散 显式声明工具依赖
可组合性 单一对话 可串联成工作流
错误处理 靠用户手动纠正 内置回退和边界条件
输出格式 自由文本 结构化输出规范

Skill 开发四步走

从 C4 系列挑战中提炼出的 Skill 开发方法论——拿来 → 改造 → 创造 → 迭代。这不是线性的，而是螺旋上升的循环。

从 AG2 或社区复制现有 skill，跑通流程
→
🔧

📥 第一步：拿来（复制现有 Skill）

目标：不要从零开始。找到已有的 skill，让它在你的环境中跑通。

for the ag2 challenge, use the skills to generate code: Plug in Beta agent skills. Copy .agents/skills/ into your repo, then open it in Cursor, Claude Code, Codex, Gemini CLI, or OpenCode. 15 AG2 Beta skills load automatically.

Richard · Elite20 Builder Program · AG2 挑战

具体操作：

  1. 克隆或下载 AG2 官方 skill 仓库：https://github.com/ag2ai/build-with-ag2
  2. 将 .agents/skills/ 目录复制到你的项目根目录
  3. 用支持 skill 的 Agent 工具（Cursor、Claude Code、Codex、Gemini CLI、OpenCode）打开项目
  4. 观察 15 个 Beta skills 是否自动加载
💡

不要追求第一次就理解全部。先跑通，再研究。跑通本身就是最好的学习。

🔧 第二步：改造（调整适配）

目标：在已有 skill 的基础上，进行参数化和定制化改造，让它适配你的具体需求。

改造的常见维度：

  • 语言调整：将英文 skill 改为中文，或反之
  • 工具替换：将特定 API 调用替换为你自己的工具链
  • 输出格式：调整输出为 Markdown、JSON、HTML 等
  • 边界条件：增加/减少错误处理和回退逻辑
  • 角色定制：修改 skill 的角色定义，让它更贴合你的领域

示例：将 AG2 的"代码审查"skill 改造为"中文作业评审"skill

✨ 第三步：创造（元技能生成）

目标：用元技能（meta-prompt）生成全新的 skill，解决你自己的真实问题。

这是 Richard 最强调的能力：用元技能生成技能。你不需要手动写 skill 的每一行——你只需要：

  1. 清楚地描述你要解决什么问题
  2. 用自然语言告诉你的 Agent（或一个"元技能"）
  3. 让 AI 为你生成 skill 的结构化定义
  4. 验证和微调
# 元技能 prompt 示例 "你是一个 Agent Skill 设计师。请为以下任务生成一个 skill： 任务：[在这里描述你的任务] 需要的工具：[列出可用工具] 输出格式：[期望的输出格式] 约束条件：[边界条件] 请输出 skill.yaml 和 instructions.md 的内容。"

🔄 第四步：迭代（持续进化）

目标：在实际使用中不断优化 skill，让它越来越准确、稳定。

Skill 不是一次写好的——它需要像软件一样迭代：

  • 收集失败案例：当 skill 出错时，记录输入和错误输出
  • 添加边界条件：基于失败案例，增加异常处理
  • 简化指令：删除冗余的提示，让 skill 更聚焦
  • 版本管理：用 git 管理 skill 的版本历史
  • 分享回馈：将你优化的 skill 分享给社区

C4 系列挑战详解

C4 系列是 Elite20 Builder Program 的四个实战挑战，每个挑战都要求你构建一个 Agent Skill。理解它们就是理解 NEOLAF + Skill 的最佳方式。

C4A · 评估器

📊 班级评估系统

扫描群文件夹 → 识别 C4 提交 → 检查完整性 → 评审质量 → 生成报告。

四级挑战：
Level 1：文件采集 — 扫描群文件夹，识别 C4 提交
Level 2：完整性检查 — 检查提交是否完整
Level 3：质量评审 — 对作业进行质量评估
Level 4：班级报告 — 生成班级整体报告
C4B · 发布器

📝 公众号发布系统

从本地文件自动排版为公众号文章，支持 Markdown 到公众号格式的转换。

核心能力：
Markdown → 富文本转换 · 自动排版
图片上传 · 格式标准化
一键发布工作流
C4C · 解题器

🧮 中文解题流水线

5步流水线：摄入 → 解析 → 求解 → 排版 → 编译。核心要求：用国产模型替代 Claude Code。

5步流水线：
1. 摄入题目（图片/文本）
2. 解析题目（理解题意）
3. 求解（用 Qwen 3.6 / Kimi 2.5）
4. 排版输出
5. 编译验证
C4D · 本地模型

💻 纯本地驱动

在本地设备下载运行 Gemma 4，用本地模型驱动 Agent，完全不依赖云端 API。

关键技术：
Gemma 4 本地部署 · Ollama/LM Studio
本地模型调用 · 离线 Agent 工作流
隐私优先 · 无网络依赖
🎯

学习建议：不要同时开始四个挑战。从你最感兴趣的一个开始（很多人从 C4C 开始），跑通一个 skill 的完整开发流程，再扩展到其他挑战。

⚡ 练习二：用元技能生成新 Skill

这是 Richard 最强调的练习——用元技能（meta-prompt）生成一个全新的 skill，而不是手动编写。

1 定义你的问题

想一个你日常工作中经常重复、但又没有系统化的事情。用一句话描述它。

我的问题：________________________________________

2 编写元技能 Prompt

用下面的模板生成你的 skill：

"你是一个 Agent Skill 设计师。请为以下任务生成一个完整的 skill： 任务描述：[在上一步定义的问题] 需要的工具：[列出你可以使用的工具] 输出格式：Markdown 约束条件：[任何特殊要求] 请输出两个文件： 1. skill.yaml — 元数据 2. instructions.md — 核心指令"

3 测试并迭代

在 Agent 中加载你生成的 skill，跑一个实际用例。记录错误和不足，调整指令，再次测试。重复直到满意。
