KSTAR 学习循环

从课堂实录到动手实操 — Richard Tong 佟博士的教学精华，你可以在每次任务中使用的框架
📚 15 条课堂引用 ⏱ 预计 45 分钟 🏋️ 4 个练习

01 课前准备

在学习 KSTAR 之前，确认你已经具备这些基础：

能区分"想法"和"行动"
有过至少一次"计划→执行→回顾"的经验
打开一个文本编辑器或准备一张纸
💡 如果还没准备好

先别急。KSTAR 不是一个需要"学懂"的概念，而是一个需要"用会"的工具。后面的每一步都会带着你做。

03 KSTAR 详解

K → S → T → A → R
Knowledge ←—— Reflection（闭环）

五步详解

Step 1
🧠 Knowledge — 知识检索

做什么：开始任何工作之前，先问自己"我已经知道什么？"

具体操作：

  • 查历史记录：以前做过类似的吗？
  • 搜索可用资源：有没有现成的代码/tool/skill？
  • 问人/问 AI：先获取情报，不要蒙头造轮子
  • 反面：上来就干，做了半天发现别人已经做过了
Step 2
📍 Situation — 情境评估

做什么：搞清楚你现在在什么位置。

具体操作：用 5W1H 问自己：

  • What — 要完成什么？具体目标是什么？
  • Why — 为什么要做？优先级有多高？
  • Where — 在什么环境里做？有什么工具？
  • When — 截止时间？现在做到哪了？
  • Who — 涉及谁？需要谁配合？
  • How — 怎么做？用什么方法？
Step 3
📋 Task — 任务拆解

做什么：把模糊目标变成可执行的子任务。

具体操作：

  • 一个大任务拆成 3-7 个小任务
  • 每个小任务必须可验证（做完后怎么知道做完了？）
  • 标注优先级：P0（必须做）> P1（重要）> P2（可选）
  • 关键：如果某个任务你还不知道怎么下手，说明它还不够小——继续拆
Step 4
⚡ Action — 执行

做什么：一次只做一件事，做完整验证。

具体操作：

  • 从优先级最高的 P0 开始
  • 不要同时做多件事
  • 做完立刻验证（测试/预览/检查）
  • 卡住了？先记录卡点，跳到下一个可做的任务
Step 5 — 最关键
🔄 Reflection — 反思

做什么：完成一个闭环后，停下来问自己五个问题。

具体操作：

  • 做对了什么？→ 记下来，下次复用
  • 踩了什么坑？→ 记下来，下次避免
  • 有什么模式？→ 可以做成 skill 吗？
  • 用户/老师的偏好？→ 更新记忆
  • 下次怎么改进？→ 带着新知识进入下一轮 K
🚫 最常见的两个错误

错误1：把 S 写成 Skill。S 是 Situation（情境），不是技能。技能是你 Action 时用的工具。

错误2：把 R 写成 Report（报告）。R 是 Reflection（反思），不是写一个文档交差——是真的在想"这次学到了什么"。

04 实操：用 KSTAR 完成一个真实任务

下面我会带你用 KSTAR 走一遍——就拿"用 wx-cli 查一下 AI+X 实验班的聊天记录"这个你已经做过的事为例：

完整示例

🟦 Knowledge 阶段
我知道什么？
- 我有 wx-cli 安装好了（之前 Richard 的课上学过）
- 我知道 AI+X 实验班1 群的群名
- 之前张浩做过 wechat-class-manager
- 我有 sudo 权限

还需要查什么？
- wx export 命令怎么用 → 查 --help
- 群的确切名字 → wx sessions 确认
🟪 Situation 阶段
当前状态：
- 微信在运行，已登录
- wx-cli 已初始化（18 个数据库密钥已提取）
- 目标：导出整个群的聊天记录
- 限制：网络被 GFW 限制，但 wx-cli 是本地工具，不影响
- 时间：不紧急，但要确保导出完整（可能 500+ 条）
🟡 Task 阶段
拆解为子任务：
P0: 确认群名全称 → wx sessions
P0: 导出为 markdown → wx export "群名" --format markdown --limit 2000
P1: 检查导出完整性 → wc -l 导出文件
P1: 筛选有用信息 → 写 python 脚本
P2: 保存到工作空间
🟢 Action 阶段
执行记录：
✓ wx sessions → 找到群名正确
✓ wx export → 导出成功，761 条消息
✓ 检查完整性 → 无截断
✓ 筛选 → 去除了系统消息和纯表情
✓ 保存到 ~/hermes工作空间/会话记录/

卡点：筛选脚本第一次没有正确处理系统消息 → 重新检查逻辑后修复
🩷 Reflection 阶段
做对了：
- 先用 wx sessions 确认群名（不要猜）
- 直接用文件输出（避免终端截断）

踩的坑：
- 群名中有特殊字符（空格），需要引号包裹
- 第一次筛选脚本没有处理好群名作为发送者的系统消息

下次可以更好：
- 做一个通用的"微信群数据导出+筛选" skill
- 这样下次 5 分钟就能完成

新 Knowledge（进入下一轮）：
- 知道了 "Elite20 Builder Program1群" 有 761 条消息
- 知道了导出+筛选的最佳流程

🏋️ 练习 2：找错 10分钟

下面这个同学的 KSTAR 作业有什么问题？

K：Hermes Agent
S：我会用 Python
T：完成 C5 挑战
A：写代码
R：写了一份报告
查看答案

多个问题：

  • K 只写了"Hermes Agent"——这不是知识，这是一个工具名。你需要写"我知道什么"（比如：我知道 Hermes Agent 有 delegate_task 功能、有 memory 系统等）
  • S 写的是技能（Skill）而不是情境（Situation）。S 应该是：当前在哪个挑战？有什么工具可用？有什么限制？
  • T "完成 C5 挑战" 不是可执行的任务——太模糊。应该拆成：① 下载 starter ② 跑通 demo ③ 修改参数 ④ 验证输出
  • A "写代码" 太笼统——你具体写了什么代码？结果如何？
  • R "写了一份报告"——这是 Report 不是 Reflection。应该反思：哪些方法有效？哪些代码还可以优化？

🏋️ 练习 3：快速 KSTAR 迭代 5分钟 × 3轮

选一个很小的事——比如"整理今天的笔记"。

走第一轮 KSTAR（5分钟）→ 做完 → Reflection → 第二轮 KSTAR（5分钟，基于第一轮的 R）→ 做完 → Reflection → 第三轮。

你不需要把每轮的 KSTAR 写得很长。关键是：每轮都比上一轮快。

🏋️ 练习 4：教别人 15分钟

找一个没上过这个课的同学（或者用 ChatGPT 扮演一个），用你自己的话解释：

  1. KSTAR 是哪五步？
  2. S 为什么不是 Skill？
  3. 为什么"快速迭代 > 完美首发"是核心？

如果你能够让别人听懂，说明你真的理解了。
