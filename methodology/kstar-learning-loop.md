# KSTAR Learning Loop

## Purpose

KSTAR explains how the system learns from every execution.

For Elite20 Builder Program / NSEAP, each Challenge submission, review, failure, and improvement should become learning material for the next cycle.

## Loop

```text
Learn What
-> What
-> So What
-> Now What
-> Execute
-> Evaluate
-> Reflect
-> Update Knowledge
-> Improve Skills
```

## In Builder OS

| Step | Meaning |
|---|---|
| Learn What | identify the target problem or Challenge |
| What | describe what happened |
| So What | explain why it matters |
| Now What | decide the next action |
| Execute | build or revise the artifact |
| Evaluate | review with Rubric and Agent |
| Reflect | write AAR and lessons |
| Update Knowledge | add FAQ, examples, patterns |
| Improve Skills | update prompts, workflows, or Skill specs |

## Knowledge Growth Outputs

Each Challenge should produce at least one reusable output:

- FAQ update
- common mistake
- strong example
- reusable prompt
- Skill improvement
- Agent manifest update
- ontology update
