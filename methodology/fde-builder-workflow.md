# FDE Builder Workflow

## Purpose

FDE means turning a real situation into a deployable AI solution.

In Elite20 Phase II, every Builder should practice this workflow:

```text
Situation
-> Ontology
-> Workflow
-> Skill
-> Task Agent
-> Evaluation
-> Release
```

## Builder Workflow

| Stage | Builder Action | Output |
|---|---|---|
| Situation | describe the real problem | situation statement |
| Context | list required information | context checklist |
| Ontology | define concepts and relationships | ontology mapping |
| Workflow | define execution steps | workflow diagram or list |
| Skill | package reusable capability | Skill spec |
| Agent | assemble capabilities into a role | Agent manifest |
| Evaluation | test with examples | evaluation report |
| Release | document and publish | README, release notes, knowledge entry |

## Relationship With GitHub

GitHub is the execution space for the FDE workflow.

Use GitHub to store:

- Situation documents
- Challenge definitions
- ontology mappings
- Skill specs
- Agent manifests
- code and artifacts
- review reports
- release notes

## Minimum FDE Submission

Every advanced Builder submission should include:

```text
README.md
situation.md
workflow.md
skill.md or skills/
agent-manifest.yaml if relevant
evaluation-report.md
reflection.md
```
