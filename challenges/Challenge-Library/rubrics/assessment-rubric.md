# 评估体系

> Challenge Library 的四维评估框架

## 一、评估理念

本评估体系遵循AI+X实验班的核心教学理念：

> **参与 > 成败，迭代 > 完美，协作 > 单干，可复用 > 一次性**

评估不是"打分"，而是**给反馈、促成长**。每个挑战的评估都包含三层：

```
┌─────────────────────────────────┐
│  ① AI Agent自动评审（即时）     │
├─────────────────────────────────┤
│  ② 同侪互评（48小时内）         │
├─────────────────────────────────┤
│  ③ 教师/企业评审（最终定级）    │
└─────────────────────────────────┘
```

## 二、四维评估框架

### 维度1: 🧠 拿来主义（权重20%）

| 级别 | 表现 | 分数 |
|------|------|------|
| ❌ 未达标 | 从零开始，没有参考任何已有方案 | 0-30% |
| ✅ 通过 | 参考了至少1个已有方案或示例 | 60% |
| 🚀 良好 | 参考多个方案并做了对比选择 | 80% |
| 🏆 优秀 | 借鉴精华并做了原创性改进 | 100% |

**评审问题**：
- 你的方案参考了哪些已有工作？
- 你是如何选择参考方案的？
- 在借鉴的基础上做了哪些改进？

### 维度2: 💬 有效反馈（权重20%）

| 级别 | 表现 | 分数 |
|------|------|------|
| ❌ 未达标 | 没有获取任何外部反馈 | 0-30% |
| ✅ 通过 | 获得至少1人的反馈 | 60% |
| 🚀 良好 | 获得2+人反馈并做了吸收 | 80% |
| 🏆 优秀 | 主动迭代反馈循环（提反馈→修改→再反馈） | 100% |

**评审问题**：
- 你向谁征求了反馈？
- 反馈意见是什么？你采纳了哪些？
- 反馈→修改→再反馈的循环进行了几次？

### 维度3: 🔄 多次迭代（权重30%）

| 级别 | 表现 | 分数 |
|------|------|------|
| ❌ 未达标 | 一版定稿，没有修改 | 0-30% |
| ✅ 通过 | 完成1轮迭代 | 60% |
| 🚀 良好 | 完成2轮迭代，每轮有明确方向 | 80% |
| 🏆 优秀 | 3轮以上迭代，有明显质量提升 | 100% |

**评审问题**：
- 从V1到最终版本，改了什么？
- 每次迭代的改进方向是什么？
- 能否展示迭代前后的对比？

### 维度4: 📦 可复用性（权重30%）

| 级别 | 表现 | 分数 |
|------|------|------|
| ❌ 未达标 | 只有自己能看懂/使用 | 0-30% |
| ✅ 通过 | 别人能理解你做了什么 | 60% |
| 🚀 良好 | 别人能直接使用你的产出 | 80% |
| 🏆 优秀 | 别人能基于你的产出继续开发 | 100% |

**评审问题**：
- 你的产出有文档吗？
- 一个陌生人能否照着你的文档跑起来？
- 你的代码/配置是否模块化、可扩展？

### 总分计算

```
总分 = 拿来主义×20% + 有效反馈×20% + 多次迭代×30% + 可复用性×30%
```

| 等级 | 分数范围 | 含义 |
|------|---------|------|
| ⭐ 通过 | 60-79分 | 已掌握基础能力 |
| ⭐⭐ 良好 | 80-89分 | 具备较强的实践能力 |
| ⭐⭐⭐ 优秀 | 90-100分 | 达到可交付水平 |

## 三、AI Agent自动评审

### 评审Agent配置

每个挑战提交后，Evaluation Agent会自动进行：

1. **完整性检查**：是否提交了所有要求的交付物
2. **格式检查**：是否符合要求的格式和命名规范
3. **Rubric初评**：基于四维框架给出初步评分
4. **改进建议**：针对每个维度的具体改进方向

### 评审Agent Prompt模板

```
你是一个Challenge评审Agent。请根据以下Rubric对提交的挑战进行评审：

挑战: [挑战名称]
提交者: [姓名]
交付物:
- [产物1]: [内容摘要]
- [产物2]: [内容摘要]
...

请按以下维度评分（每项0-100分），并给出评语和改进建议：
1. 拿来主义
2. 有效反馈
3. 多次迭代
4. 可复用性
```

## 四、同侪互评

### 互评流程

```
提交截止 → 随机分配3位同学互评 →
48小时内完成互评 → 汇总反馈 →
根据反馈进行最终修改 → 提交终版
```

### 互评模板


## 五、教师/企业评审

教师评审用于最终定级，重点关注：

1. **真实问题解决** — 是否解决了真实世界的问题
2. **产品思维** — 是否具备产品化思维（用户体验、文档、部署）
3. **成长轨迹** — 从第一次提交到最终版本的进步幅度
4. **展示能力** — 路演/演示的表达质量
