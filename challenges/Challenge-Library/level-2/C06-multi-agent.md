# C06 · 多智能体协作系统

> 难度：L2（进阶） | 类型：构建型 | 预估耗时：5-7天

## 📋 挑战概述

构建一个由多个智能体组成的协作系统。每个智能体负责不同的角色和任务，它们通过消息传递协同完成复杂工作。你将体验"AI团队"的设计和搭建。

> **核心理念**：单个智能体有极限，多个智能体协作能解决更复杂的问题。你不仅是开发者，更是"AI团队的架构师"。

### 前置要求
- 完成C05（单智能体系统开发）
- 理解Agent的基本架构
- 了解多进程/多线程基本概念

## 🎯 学习目标

完成本挑战后，你将能够：

1. 设计多智能体系统的角色分工
2. 实现智能体之间的通信和协作机制
3. 协调多个智能体完成复杂任务
4. 诊断和优化多智能体系统的性能

## 🧩 挑战步骤

### Step 1: 设计多智能体系统（1天）

选择一个应用场景，设计智能体团队：

| 场景 | 建议角色 |
|------|---------|
| 📝 **文章创作** | 策划Agent + 写作Agent + 编辑Agent + 插图Agent |
| 💻 **软件开发** | PM Agent + 开发者Agent + 测试Agent + 文档Agent |
| 📊 **数据分析** | 数据采集Agent + 分析Agent + 报告Agent + 可视化Agent |
| 🎓 **在线教学** | 讲师Agent + 助教Agent + 评测Agent + 答疑Agent |
| 🛠️ **客户服务** | 接待Agent + 技术Agent + 升级Agent + 反馈Agent |

**你的任务**：向AI描述你选的场景，让它帮你完善分工设计。

> "我要做一个多智能体文章创作系统。请帮我设计角色分工、通信协议和协作流程。输出系统架构图（可用文字描述）。"

### Step 2: 搭建基础通信框架（1天）

用AG2（AutoGen）或Hermes Agent搭建多智能体框架：

**方案A: AG2 (推荐)**
```python
# 核心概念：Agent之间通过消息通信
assistant = AssistantAgent(name="助手", ...)
user_proxy = UserProxyAgent(name="用户", ...)
# 建立对话流程
user_proxy.initiate_chat(assistant, message="...")
```

**方案B: Hermes Agent**
- 用多个Skill对应多个Agent角色
- 通过delegate_task进行任务分发

**你的任务**：先让至少2个智能体能互相通信，打印出消息日志。

### Step 3: 实现角色功能（2天）

为每个智能体实现核心功能：

| 步骤 | 内容 | 验收标准 |
|------|------|---------|
| 1. 角色定义 | 编写每个Agent的System Prompt | 角色清晰，不重叠 |
| 2. 工具分配 | 给每个Agent分配需要的工具 | 工具与角色匹配 |
| 3. 通信测试 | Agent之间能正常收发消息 | 消息不丢失、不混乱 |
| 4. 流程验证 | 完整跑通一个任务 | 从开始到结束，结果正确 |

### Step 4: 调试与优化（1-2天）

典型问题及调试方法：

| 问题 | 症状 | 解决方案 |
|------|------|---------|
| Agent之间无限对话 | 不停地来回发消息 | 设置最大对话轮次 |
| Agent角色混淆 | 做其他Agent的事 | 优化System Prompt |
| 任务丢失 | 某个步骤没执行 | 增加确认机制 |
| 输出质量差 | 结果不符合预期 | 增加评审Agent或人工干预 |

### Step 5: 演示与AAR

完整运行一次系统，记录所有Agent的对话日志。

## 💡 提示与常见坑

- ✅ **先让2个Agent跑通**，再逐步增加
- ✅ **给每个Agent明确的"边界"**——它该做什么、不该做什么
- 🚫 **不要让所有Agent都能调用所有工具**——权限最小化
- ✅ **加一个"协调者Agent"**来管理任务分配和结果汇总
- 🚫 **注意Token消耗**——多Agent对话会消耗大量Token，设置合理的对话轮次上限
