# C08 · 飞书/钉钉/微信集成

> 难度：L2（进阶） | 类型：产品型 | 预估耗时：5-7天

## 📋 挑战概述

让你的AI智能体能接入**飞书、钉钉或微信**——在国民级IM应用中运行。你将学会搭建"IM ↔ AI Agent"的桥梁，让智能体在聊天中为用户提供服务。

> **核心理念**：AI的能力最终要在用户日常使用的地方落地。IM就是最好的入口。

### 前置要求
- 完成C05（单智能体系统开发）
- 有基本的网络编程概念（Webhook、API）
- 完成C08不需要微信/钉钉/飞书的所有权限——选择其中一个即可

## 🎯 学习目标

完成本挑战后，你将能够：

1. 理解IM Bot的工作原理（Webhook + API）
2. 搭建一个飞书/钉钉/微信机器人
3. 让机器人能接收消息并调用AI处理
4. 让机器人回复用户消息

## 🧩 挑战步骤

### Step 1: 选择平台（半天）

选一个你常用的平台：

| 平台 | 优势 | 挑战 | 推荐度 |
|------|------|------|--------|
| **飞书** 🥇 | API开放、文档完善、有沙箱环境 | 需要飞书开发者账号 | ⭐⭐⭐⭐⭐ |
| **钉钉** 🥈 | 企业用户多、API丰富 | 需要企业认证 | ⭐⭐⭐⭐ |
| **企业微信** 🥉 | 与微信互通 | 需要企业认证 | ⭐⭐⭐ |
| **微信个人号** | 个人用户多 | 有封号风险（不建议） | ⭐ |
| **Telegram** 🌍 | API最开放 | 国内访问受限 | ⭐⭐⭐ |

**推荐：飞书**（最易上手）

### Step 2: 注册Bot（1天）

**飞书流程**：
1. 注册飞书开发者账号 (open.feishu.cn)
2. 创建应用 → 启用机器人能力
3. 配置权限（消息读取、消息发送）
4. 获取App ID和App Secret
5. 配置Event Subscription（接收消息的URL）

**你的任务**：让AI帮你完成配置。

> "我要在飞书创建一个机器人。请一步步告诉我怎么在飞书开放平台创建应用、开启机器人能力、配置事件订阅。我已经有了飞书开发者账号。"

### Step 3: 搭建Bot服务（2天）

用Hermes Gateway或自己写一个Web服务：

**方案A: 用Hermes Gateway (推荐)**
配置config.yaml中的gateway部分，添加飞书平台。

**方案B: 自己写**
```python
from flask import Flask, request
# 接收飞书Webhook
@app.route('/webhook', methods=['POST'])
def webhook():
    data = request.json
    # 调用AI处理
    # 回复消息
    return 'OK'
```

**关键功能**：
1. 接收消息 → 解析消息内容
2. 调用AI处理 → 生成回复
3. 发送回复 → 返回给用户

### Step 4: 功能增强（1-2天）

在基础对话之外，增加功能：

| 功能 | 实现方式 | 难度 |
|------|---------|------|
| @机器人提问 | 解析@消息 | ⭐⭐ |
| 关键词触发 | 配置关键词规则 | ⭐⭐ |
| 定时推送 | 使用Cron任务 | ⭐⭐⭐ |
| 图片识别 | 调用视觉API | ⭐⭐⭐ |
| 群聊管理 | 统计/提醒/降噪 | ⭐⭐⭐⭐ |

> "我的机器人已经能回复消息了。现在我想添加'每日AI早报'功能，每天早上8点自动推送AI领域的最新动态。请告诉我怎么实现。"

## 💡 提示与常见坑

- ✅ **先用飞书**——开发体验最好，有沙箱环境
- ✅ **用Webhook.site测试回调**——先确保你的服务能被外部访问
- 🚫 **不要尝试微信个人号**——封号风险高，不值得
- 🚫 **注意消息频率限制**——飞书/钉钉都有API限流
- ✅ **让AI帮你写Webhook处理函数**——这是机械工作，AI很擅长
