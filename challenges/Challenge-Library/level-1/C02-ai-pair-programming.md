# C02 · AI结对编程入门

> 难度：L1（入门） | 类型：探索型 | 预估耗时：2-3天

## 📋 挑战概述

体验AI辅助编程的完整流程——不是"让AI替你写代码"，而是"让AI做你的结对编程伙伴"。你设计逻辑，AI写代码，你评审结果，你们一起改。

> **核心理念**：编程正在从"写代码"变成"设计解决方案"。你的价值在思路和判断，不在打字速度。

### 前置要求
- 了解编程的基本概念（变量、循环、函数），不需要精通
- 会使用一个AI编程工具（Claude Code、Codex、Cursor、Hermes等）
- 本地安装Python或Node.js运行环境

## 🎯 学习目标

完成本挑战后，你将能够：

1. 理解"AI结对编程"的工作模式——你说思路，AI写代码
2. 用自然语言描述功能需求，让AI生成可运行的代码
3. 阅读和理解AI生成的代码
4. 对AI输出进行测试、调试和迭代

## 📦 交付物

| 产物 | 说明 | 最低标准 |
|------|------|---------|
| ① 需求文档 | 你要做的程序是什么、有什么功能 | 能用自然语言说清楚 |
| ② 可运行程序 | AI协助完成的代码 | 能正常运行 |
| ③ 测试记录 | 你测试了什么、发现了什么bug、怎么修的 | 至少3条测试用例 |
| ④ AAR复盘 | 课后复盘 | 包含协作体验和改进方向 |

## 🧩 挑战步骤

### Step 1: 选一个项目（半天）

从以下选一个，或者自定义：

| 选项 | 难度 | 说明 |
|------|------|------|
| TODO List管理器 | ⭐ | 增加/删除/标记完成 |
| 天气查询小程序 | ⭐⭐ | 调用API显示天气 |
| 个人记账本 | ⭐⭐ | 记录收支，生成统计 |
| 学习卡片APP | ⭐⭐⭐ | 卡片式复习工具 |

**你的任务**：向AI描述你选的项目，让它帮你完善需求。

> "我想做一个TODO List管理器。用户能添加任务、标记完成、删除任务。数据存储在本地，下次打开还在。请帮我细化功能需求。"

### Step 2: AI生成初版代码（半天）

把你细化的需求给AI：

> "基于上面的需求，请生成一个完整的Python程序。用命令行交互方式。功能：1. 添加任务 2. 查看所有任务 3. 标记完成 4. 删除任务。数据用JSON文件保存。"

**你的任务**：
1. 运行AI生成的代码
2. 测试所有功能
3. 如果报错，把错误信息贴回给AI让它修

### Step 3: 测试与迭代（1天）

创建一张**测试记录表**：

| 测试用例 | 预期结果 | 实际结果 | 是否通过 |
|---------|---------|---------|---------|
| 添加一个任务"买书" | 任务列表显示"买书" | | |
| 标记任务完成 | 状态变为"已完成" | | |
| 删除任务 | 任务从列表消失 | | |
| 重复添加相同任务 | 允许/拒绝？ | | |
| 程序关闭再打开 | 数据还在 | | |

把发现的bug告诉AI，让它修。**至少迭代2轮**。

### Step 4: 增强功能（可选，半天）

完成基础版后，选择一个增强方向：

- 增加优先级（高/中/低）
- 增加截止日期
- 增加分类/标签
- 改为图形界面（用HTML或Tkinter）

> "我已经有了命令行版本。请把它改成Web版，用HTML+CSS+JavaScript做一个好看的页面。"

### Step 5: AAR复盘

## 📐 评分标准（Rubric）

| 维度 | 通过 (60%) | 良好 (80%) | 优秀 (100%) |
|------|-----------|-----------|-----------|
| **功能完整性** | 核心功能可用 | 核心功能+边界处理 | 核心功能+增强功能 |
| **代码质量** | 能运行 | 代码结构清晰，有注释 | 代码可复用、有错误处理 |
| **测试充分性** | 有测试记录 | 覆盖核心功能的测试 | 覆盖边界情况的测试 |
| **迭代深度** | 完成1轮迭代 | 完成2轮迭代 | 迭代有明确方向和改进 |
| **AAR质量** | 完成AAR | 有具体的协作体验描述 | 可操作的改进计划 |

## 💡 提示与常见坑

- ✅ **先把思路讲清楚再让AI写**，而不是让AI从零猜
- ✅ **每次只让AI做一件事**：先写功能，再写UI，再写测试
- 🚫 **不要提交AI一次生成的代码**——它一定有可以改进的地方
- 🚫 **不要复制粘贴不理解的代码**——让AI逐行解释
- ✅ **遇到bug先自己猜原因，再让AI确认**——这是最好的学习方式
