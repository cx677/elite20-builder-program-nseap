# KSTAR 情境方案模板

> 用于在挑战中提出具体改进方案时使用

## 模板结构

```
K — 知识（Knowledge）: 引用的理论、论文、已有工作
S — 情境（Situation）: 当前现状和背景
T — 任务（Task）: 要解决的具体问题
Â — 行动（Action）: 具体的执行方案
R̂ — 预期结果（Expected Result）: 可观测的改进证据
```

---

## 填写示例

```
K — 知识
AI+X论文6.3节指出，Elite20计划的第三阶段需要建立"数字留痕"机制，
但目前主要依赖学生自发上传，缺乏系统性。

S — 情境
目前实验班学生的项目提交分散在微信群、飞书、GitHub等多个平台，
班主任需要手动收集和统计，耗时且容易遗漏。

T — 任务
设计一个统一的项目提交与追踪系统，自动聚合多平台提交通知，
生成实时进度看板。

Â — 行动
1. 使用飞书多维表格作为统一提交入口
2. 配置Webhook自动抓取提交通知
3. 用Hermes Agent定时生成进度报告
4. 每周自动推送给班主任和佟博士

R̂ — 预期结果
1. 提交统计时间从人工2小时减至自动0小时
2. 项目进度透明化，延迟项目能被及时发现
3. 本方案可在1周内部署完成
```
